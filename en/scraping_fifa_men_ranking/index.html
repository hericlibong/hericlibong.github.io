Scraping the Fifa men's ranking with Scrapy and the hidden API

Heric Libong included in Scraping
 2023-04-08  1415 words   7 minutes 
Contents
    1. Find Hidden API
    2. Get data with Scrapy

After the publication of the latest FIFA ranking on April 6th, I visited the association’s website to examine their procedures and potentially obtain the historical ranking since its creation in 1992. While I had access to their procedures, I could not find any public API or an easy way to download the data from their tables. Consequently, I decided to collect the data myself.

Find Hidden API

When it comes to extracting data from a dynamic website, my first approach is to search for a hidden API. Not only does this allow me to avoid getting blocked, but it also enables me to quickly obtain structured data without relying on long and complex code. Although not all sites have a hidden API, finding one that allows access to the data can greatly simplify the task.

So, I immediately opened the browser inspector and… Bingo! After refreshing the page, I found an API URL in the XHR tab, which returns the complete current ranking data in a JSON format list.

If we perform the same process by selecting a different date from the drop-down menu, we observe that the page consistently returns the same type of data. Our objective, therefore, will be to retrieve all of these pages. But for now, let’s focus on how to retrieve the latest ranking data using Scrapy.

Get data with Scrapy

have chosen to use Scrapy because it is my preferred scraping library and it provides a powerful and flexible framework for web scraping tasks. For this project, it can be used outside of its environment, from a simple file, but to speed up the process, we will create a Scrapy project with the default template spider package :

1pip install scrapy

1scrapy startproject <projectname>

In the spider folder : 
1scrapy genspider <filename> <site.com>

The first objective is to extract the data from the HTTP response and return it as a list, as we see it on the API endpoint link. Since the URL returns JSON data, we need to:

  • Import the json library
  • Enter the correct domain name: fifa.com
  • Enter the endpoint URL into the starts_url variable
  • In the parse function, load the content of the HTTP response into the data variable as a Python dictionary
  • Loop through the rankings of all teams and return the data.

 1import scrapy
 2import json  #The URL returns data in JSON format.
 3
 4
 5class MenrankingSpider(scrapy.Spider):
 6    name = 'menranking'
 7    allowed_domains = ['fifa.com']
 8    start_urls = ['https://www.fifa.com/api/ranking-overview?locale=en&dateId=id13974']
 9
10    def parse(self, response):
11        data = json.loads(response.body) #loads the contents of the HTTP response into the `data` variable as a Python dictionary.
12        for ranking_data in data['rankings']:
13            yield ranking_data

In the terminal

1scrapy runspider filename.py

The code instantly returns all the data from the endpoint.

From there, we can select the data we want to keep for exporting in different formats. For me, I only need:

  • the country name of the team
  • the total points
  • the previous points
  • the rank
  • the previous rank
  • the flag link
  • the country URL
  • the confederation.

 1import scrapy
 2import json  #The URL returns data in JSON format.
 3
 4class MenrankingSpider(scrapy.Spider):
 5    name = 'menranking'
 6    allowed_domains = ['fifa.com']
 7    start_urls = ['https://www.fifa.com/api/ranking-overview?locale=en&dateId=id13974']
 8
 9    def parse(self, response):
10        data = json.loads(response.body) #loads the contents of the HTTP response into the `data` variable as a Python dictionary.
11        base_url = 'https://www.fifa.com'
12        for ranking_data in data['rankings']:
13            yield {
14                'country': ranking_data['rankingItem']['name'],
15                'rank': ranking_data['rankingItem']['rank'],
16                'previousRank' : ranking_data['rankingItem']['previousRank'],
17                'totalPoints': ranking_data['rankingItem']['totalPoints'],
18                'previousPoints': ranking_data['previousPoints'],
19                'flagUrl': ranking_data['rankingItem']['flag']['src'],
20                'countryUrl': base_url + ranking_data['rankingItem']['countryURL'],
21                'conf': ranking_data['tag']['text'] 
22            }

Again, the data is returned instantly. However, we notice that we do not have the publication date of the ranking. We will see later how to fix this.

Get all data from all pages

The main challenge when extracting data from a hidden API is to find the keys that allow you to navigate from one page to another. These are usually found in the metadata, at the beginning or sometimes at the end of the list of data returned by the endpoint, and vary depending on the API.

It can be a page number, the total number of pages, an ID number, or even a boolean indicating that there is a next page.

The issue here is that there are no metadata available. Moreover, as we mentioned earlier, no date is specified. Thus, at first glance, we do not have a key that links the ranking dates to the IDs, and hence to the different data pages.

My solution — there may be others — consisted of examining the page’s source code. In the middle of the code, it was found that the corresponding dates and identifiers were all a list of dictionaries containing a key-value pair with “id” for the ID number, and “text” for the date.

1"dates":[{"id":"id13974","text":"6 Apr 2023"},{"id":"id13869","text":"22 Dec 2022"}...{"id":"id2","text":"8 Aug 1993"},{"id":"id1","text":"31 Dec 1992"}]

We just need to extract this list, turn it into a JSON content, and write a function to send it to the API URL. We can first test this by writing the function in a separate file, for example: test.py.

 1import scrapy
 2import json
 3
 4class MySpider(scrapy.Spider):
 5    name = 'myspider'
 6    start_urls = ['https://www.fifa.com/fifa-world-ranking/men?dateId=id13974']
 7
 8    def parse(self, response):
 9        #extracts content of the first script tag in the response that contains the string `"dates"` with all the json content of the page.
10        script_content = response.xpath('//script[contains(., "dates")]/text()').extract_first()
11        #transform it into a Python dictionary
12        date_list = json.loads(script_content)
13        #extract the list of dates from the dictionary
14        date_list = date_list['props']['pageProps']['pageData']['ranking']['dates']
15        
16        print(date_list)

1
2scrapy runspider test.py

In order to extract the list of dates, I used json formatter, a free online tool that makes it easy to quickly identify the correct keys. Simply copy and paste the entire JSON content into the window and click the “Process” button.”

We can now integrate this function and modify our code to retrieve all API pages and therefore all FIFA rankings since its creation in 1992. Let’s proceed step by step:

  • Import datetime module, which will be used in later steps to work with the date format in the page’s source code.
  • Change the starting URL to the current men’s ranking page, which will be used to extract the date and ID information needed to scrape all rankings data (Note that we can choose any page because the data we are interested in is the dates and the identification number of the source code).
 1import scrapy
 2import json
 3from datetime import datetime
 4
 5
 6class MenrankingSpider(scrapy.Spider):
 7    name = 'menranking'
 8    allowed_domains = ['fifa.com']
 9    # changing the starting url by using the one of the ranking page and not the one of the API
10    start_urls = ['https://www.fifa.com/fifa-world-ranking/men?dateId=id13974']
  • Add our method for extracting the list of dates and code source IDs to the parse function
  • Create a loop with the list of dates and IDs by iterating over the API URL.
  • Format the date as ‘yyyy-mm-dd’.
  • Send the request with the date as a parameter.
 1def parse(self, response):
 2        script_content = response.xpath('//script[contains(., "dates")]/text()').extract_first()
 3        date_list = json.loads(script_content)
 4        date_list = date_list['props']['pageProps']['pageData']['ranking']['dates']
 5        for item_id in date_list:
 6            url = f"https://www.fifa.com/api/ranking-overview?locale=en&dateId={item_id['id']}"
 7            date_text = item_id['text'].replace('Sept', 'sep') # Set the date to the correct format
 8            date_obj = datetime.strptime(date_text, '%d %b %Y')
 9            date = date_obj.strftime('%Y-%m-%d')
10            yield scrapy.Request(url=url, callback=self.parse_ranking_data, meta={'url': url,'date': date})

Finally, we create a new function parse_ranking_data which is a modified version of the initial parse function, with the addition of the date as metadata for each request.

 1def parse_ranking_data(self, response):
 2        data = json.loads(response.body) 
 3        base_url = 'https://www.fifa.com'
 4        for ranking_data in data['rankings']:
 5            yield {
 6                'date': response.meta['date'],
 7                'country': ranking_data['rankingItem']['name'],
 8                'rank': ranking_data['rankingItem']['rank'],
 9                'previousRank' : ranking_data['rankingItem']['previousRank'],
10                'totalPoints': ranking_data['rankingItem']['totalPoints'],
11                'previousPoints': ranking_data['previousPoints'],
12                'flagUrl': ranking_data['rankingItem']['flag']['src'],
13                'countryUrl': base_url + ranking_data['rankingItem']['countryURL'],
14                'conf': ranking_data['tag']['text'] 
15            }

Now, we can use scrapy’s export functions to get the data in a json file.

1scrapy runspider filename.py -o data.json

or in csv

1scrapy runspider filename.py -o data.csv

And 20 secondes later


In conclusion, fetching data through a hidden API can be a convenient, efficient, and fast way to extract data. However, it’s important to note that each API is unique and can have its own quirks and drawbacks. As our example of retrieving data from the FIFA ranking shows, it is necessary to adapt to each API, and not to neglect any track in order to make the most of it and obtain the desired data.
